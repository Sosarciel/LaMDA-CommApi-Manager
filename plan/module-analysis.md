## 已验证问题

### P0 紧急修复

#### 1. SelfIdEventTable 重复检测过于严格 ✅ 已确认
**位置**: `OneBot/OneBot.ts:73`
**问题**: 使用 `SLogger.error` 但不阻止操作
**影响**: 相同消息可能触发多次处理
**验证方法**: 检查日志中是否有大量重复错误

#### 2. 消息处理异常未传播 ✅ 已确认
**位置**: `CommApiManager.ts:66-68`
**问题**: 异常被吞没，上层无法感知处理失败
**影响**: 
- 消息处理失败时无法重试
- 调试困难
- 可能导致消息丢失

---

### P1 重要改进

#### 1. modify_modules 维护成本高 ✅ 已确认
**问题**: 
- 修改了 `discord.js` 和 `node-telegram-bot-api` 源码
- 版本升级困难
- 每次库更新都需要重新应用修改

**建议方案**:
```bash
# 使用 patch-package 保存修改
npm install patch-package
# 修改后运行
npx patch-package <package-name>
```

#### 2. 平台适配器抽象 ⚠️ 需评估
**当前问题**: 每个平台独立实现，缺少抽象层

**评估**:
| 平台 | 共同点 | 差异点 |
|------|--------|--------|
| Telegram | 消息发送/接收 | API 差异大 |
| Discord | 消息发送/接收 | WebSocket 模式 |
| OneBot | 消息发送/接收 | 正向/反向 WebSocket |
| KOOK | 消息发送/接收 | API 差异 |
| QQOfficial | 消息发送/接收 | 协议复杂 |

**建议**: 评估抽象收益与复杂度成本后再决定

## 新发现的机会

### P0 新发现问题

#### 3. WebSocket 重连机制缺失
**问题**: WebSocket 连接断开后无自动重连
**影响**: 服务长时间运行后可能失去消息接收能力
**建议**: 添加指数退避重连

#### 4. 消息处理无幂等性保障
**问题**: 同一消息可能被处理多次
**影响**: 业务逻辑可能执行多次
**建议**: 使用消息 ID 进行去重

### P1 新发现机会

#### 3. 消息队列支持缺失
**问题**: 高并发时消息可能丢失
**建议**: 添加内存队列或使用消息中间件

#### 4. 发送限流缺失
**问题**: 可能触发平台限流
**建议**: 添加 token bucket 限流器

#### 5. 审计日志缺失
**问题**: 无法追踪消息处理历史
**建议**: 添加消息处理日志

### P2 新发现机会

#### 3. 健康检查接口
**建议**: 添加 `isHealthy()` 方法

#### 4. 消息回执机制
**建议**: 支持消息状态追踪
